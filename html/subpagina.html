El desarrollo del Software

El desarrollo de software se refiere al proceso de diseño, creación, prueba y mantenimiento de sistemas.

Todo este proceso se realiza mediante el código fuente que se dedica a la codificación, pruebas, despliegue y mantenimiento.

El código fuente, se maneja utilizando repositorios. Un repositorio es una ubicación centralizada donde los desarrolladores web almacenan y administran el código fuente en archivos.

Hay dos tipos de repositorios:

  • Repositorio Central
  • Repositorio Distribuido

El repositorio central (VCS) contiene una copia oficial del código fuente. Hay tres tipos:

  • Repositorio Central
  • Subversión (SVN)
  • Team Fundation Server (TFS)

El repositorio distribuido no hay un repositorio oficial y contiene una copia completa del código fuente. Hay dos tipos:

  • GIT
  • MERCURIAL

GIT

GIT es un sistema de control de versiones distribuido y de código abierto

Este tipo de sistema se caracteriza por:

  • Control de versiones distribuido
  • Los usuarios mantienen el código en máquinas locales
  • Los usuarios pueden hacer cambios locales sin necesidad de internet
  • Pull & Push sincroniza los cambios locales

GIT cuenta con repositorios que son estructuras de datos que contiene una colección de archivos, directorios y el historial de esos archivos.

Para poder publicar el código fuente en GIT, se utilizan una serie de comandos que son:

  • Git clone: Sirve para crear una copia local de un repositorio remoto.
  • Git init: Indica que hay un nuevo repositorio de git en el directorio del equipo local.
  • Git add: Sirve para agregar cambios o nuevos archivos a el área de ensayo.
  • Git commit: Indica un comentario de los cambios que se han realizado en nuestro repositorio.
  • Git push: Comando utilizado para cargar confirmados locales en nuestro repositorio.
  • Git pull: Comando utilizado para recuperar y combinar los cambios de un repositorio remoto en la rama actual.
  • Git checkaut: Comando para crear una nueva rama y cambiar en un solo paso.

También, Git cuenta con README. Un archivo de uso común en proyectos de desarrollo web, utilizado para agregar información sobre nuestra página.
